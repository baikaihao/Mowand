# Mowand（魔杖）功能需求文档

## 1. 产品概述

**Mowand（魔杖）** 是一款 macOS 鼠标手势自动化工具。用户像挥动魔杖一样绘制八方向手势，并将手势绑定到系统操作、快捷键、应用启动或自动化脚本。

首版以 GitHub 分发为目标，优先适配 macOS 26 的 Liquid Glass 与新窗口样式，同时向下兼容旧版 macOS。

## 2. 核心目标

- 让用户通过鼠标手势快速执行常用操作。
- 支持全局手势，并允许按应用单独配置。
- 使用八方向自由组合手势模型，而不是固定预设形状。
- 提供默认手势模板，用户安装后无需从零配置。
- 支持系统操作、快捷键、应用启动、脚本与动作链。
- 提供符合 macOS 26 风格的现代化设置界面。
- 通过菜单栏常驻，让 App 可后台运行。

## 3. 手势系统

### 3.1 八方向模型

Mowand 支持八个基础方向：

- 东
- 南
- 西
- 北
- 东北
- 东南
- 西南
- 西北

用户可以将任意方向自由组合成一个手势，例如：

- `东`
- `东 -> 南`
- `西北 -> 东 -> 南`
- `东南 -> 西 -> 北`

系统按方向序列识别手势，而不是按固定图形名称识别。

### 3.2 手势识别流程

识别流程：

1. 用户按住指定鼠标按钮或修饰键开始绘制。
2. 系统采样鼠标轨迹。
3. 对轨迹进行去抖、平滑和最小距离过滤。
4. 将轨迹角度归类到八方向。
5. 合并连续相同方向。
6. 应用方向切换阈值和最小长度规则。
7. 匹配对应手势。
8. 松开触发按钮后执行动作链。

### 3.3 触发条件

每个手势规则可配置触发条件：

- 鼠标按钮
- 是否按住修饰键
- 当前 App
- 屏幕区域

屏幕区域支持：

- 全屏
- 左半屏
- 右半屏
- 上半屏
- 下半屏
- 左上四分之一屏幕
- 右上四分之一屏幕
- 左下四分之一屏幕
- 右下四分之一屏幕
-  自定义范围 用户划取屏幕范围

## 4. 默认手势模板

首版内置一套默认手势模板，用户安装后可直接使用。

建议默认模板：

| 手势 | 默认动作 |
|---|---|
| 西 | 返回 |
| 东 | 前进 |
| 南 | 降低音量 |
| 北 | 提高音量  |
| 东 -> 南 | 刷新 |
| 南 -> 东 | 截图 |
| 北 -> 南 | 显示桌面 |
| 西 -> 东 | 切换最近使用的 App |
| 东北 | 提高亮度 |
| 东南 | 降低亮度 |
| 西南 | 静音 |
| 西北 | 锁屏 |

默认模板要求：

- 首次启动时自动创建。
- 用户修改后不被自动覆盖。
- 提供“恢复默认模板”入口。
- 与用户自定义手势冲突时给出提示。

## 5. 动作系统

### 5.1 动作类型

Mowand 支持以下动作类型：

- 系统操作
- 模拟快捷键
- 打开 App
- 打开 URL
- 打开文件或目录
- 等待
- 运行 AppleScript
- 运行 Shell 脚本
- 触发 Shortcuts 快捷指令

### 5.2 动作链

一个手势可以绑定多个动作步骤，并按顺序执行。

动作链支持：

- 顺序执行
- 等待延迟
- 失败中断
- 失败继续
- 执行状态反馈

示例：

- 截图 -> 保存到指定目录
- 打开 App -> 等待 0.5 秒 -> 执行快捷键
- 复制 -> 运行脚本处理剪贴板 -> 粘贴

### 5.3 脚本安全

由于支持 Shell 和 AppleScript，Mowand 需要提供安全机制：

- 脚本动作显示完整内容预览。
- 第一次运行脚本前需要用户确认。
- 导入配置中的脚本动作默认禁用。
- 危险命令显示风险提示。
- 脚本执行失败时展示错误信息。
- 不静默执行未知来源脚本。

## 6. 应用规则

Mowand 支持全局规则和应用专属规则。

规则优先级：

1. 当前 App 专属规则
2. 全局规则
3. 排除规则

应用规则能力：

- 为不同 App 设置不同手势。
- 禁用某个 App 内的全局手势。
- 为某个 App 覆盖全局手势。
- 配置排除应用列表。

示例：

- Safari 中 `西` 表示返回。
- Finder 中 `西` 表示返回上级目录。
- Xcode 中 `东 -> 南` 表示运行测试。
- 游戏或绘图软件中可禁用全局手势。

## 7. 取消机制

用户画错手势时可以取消执行。

取消方式：

- 按 `Esc`
- 超过手势超时时间 提供超时时间选择

取消反馈：

- HUD 显示“已取消”
- 不执行任何动作

## 8. 菜单栏入口

Mowand 作为后台效率工具，应提供菜单栏常驻入口。

菜单栏功能：

- 开关全局手势
- 打开设置
- 显示权限状态
- 退出 App

主窗口关闭后，Mowand 仍可在后台运行。

## 9. 主界面与 macOS 26 风格

### 9.1 窗口结构

主窗口采用类似 Xcode 26 的左侧主导布局：

- 红黄绿窗口按钮位于左侧栏顶部区域。
- 侧边栏开关和主要导航按钮集中在左上。
- sidebar 展开时，顶部工具栏与侧边栏视觉融合。
- sidebar 收起时，左侧保留窄栏入口。
- 内容区顶部显示当前页面或配置组选择器。

实现要求：

- 使用公开 API。
- 不使用私有 API。
- 目标是高度接近 macOS 26 风格，而不是像素级复刻 Xcode。
- macOS 26 使用 Liquid Glass / glass-style controls。
- 旧系统回退到标准 translucent material 和普通 toolbar 样式。

### 9.2 主要页面

主界面包含：

- 手势列表
- 手势编辑器
- 动作链编辑器(兼脚本管理)
- 应用规则
- 通用设置

### 9.3 手势 HUD

绘制手势时显示轻量 HUD。

HUD 展示：

- 鼠标轨迹
- 当前识别出的方向序列
- 匹配到的动作
- 取消状态
- 错误或冲突提示

HUD 要求：

- 不遮挡主要内容。
- 支持关闭。
- 深色/浅色模式下清晰可读。
- macOS 26 上使用轻量 Liquid Glass 风格。
- 旧系统使用普通半透明材质。

## 10. 权限

Mowand 需要按需引导用户开启权限。

必要权限：

- 辅助功能权限：用于全局鼠标监听和快捷键模拟。
- 自动化权限：用于控制其他 App 或执行 AppleScript。
- 文件访问权限：用于打开文件、保存截图、访问脚本路径。

权限体验：

- 首次启动展示权限状态。
- 未授权时明确说明受影响功能。
- 提供跳转系统设置入口。
- 权限撤销后及时提示用户。

## 11. 配置与数据

Mowand 需要本地保存用户配置。

配置内容包括：

- 手势规则
- 默认模板状态
- 应用专属规则
- 排除应用
- 动作链
- 脚本动作
- HUD 设置
- 全局开关状态
- 触发条件
- 安全确认状态

配置能力：

- 导入配置
- 导出配置
- 恢复默认模板
- 重置所有设置

导入要求：

- 检测配置版本。
- 检测手势冲突。
- 脚本动作默认禁用。
- 导入后展示摘要。

## 12. 兼容性

首版目标：

- 最佳体验：macOS 26
- 功能兼容：旧版 macOS

兼容策略：

- 使用 availability checks 判断系统能力。
- macOS 26 使用 Liquid Glass 相关视觉能力。
- 旧系统回退到标准 SwiftUI/AppKit 视觉样式。
- 不依赖私有 API。

- 屏幕区域条件。

### 13.3 动作链测试

## 14. 非目标

首版不包含：

- Mac App Store 分发适配。
- 云同步。
- 插件市场。
- 团队配置管理。
- 私有 API 实现的 Xcode 像素级窗口复刻。
- 机器学习手势识别。

## 15. 默认假设

- Mowand 面向高级 Mac 用户和效率工具用户。
- 首版以 GitHub 分发为主。
- 默认启用全局手势，但用户可以通过菜单栏一键关闭。
- 默认模板会在首次启动时创建。
- 用户修改后的配置不会被自动覆盖。
- 八方向自由组合、触发条件和动作链是首版核心能力。
- macOS 26 是最佳视觉体验目标；旧系统是功能兼容目标。
